# Part B Data — Toy Dataset

## How the dataset was obtained

The dataset is **synthetically generated** in `partB/task_2_1.ipynb`. It is **not** the paper’s original LETOR OHSUMED data (use of that dataset is forbidden for Part B).

- **Source**: `sklearn.datasets.make_classification` with fixed random seed (42).
- **Process**: We generate a pool of samples (e.g. 200) with continuous features and binary labels, then assign each sample to a synthetic **query ID** so that each “query” has a fixed number of document vectors (query–document structure). Train and test splits are made **by query** (e.g. 80% of query IDs for train, 20% for test) to avoid leakage.
- **Normalization**: `StandardScaler` is fitted on the training set only; the same scaler is applied to the test set. The scaler is saved as `scaler.npy` for reproducibility.

## How the dataset is used

- **task_2_1.ipynb**: Generates the data and saves `train_data.npy`, `test_data.npy`, and `scaler.npy` in this folder.
- **task_2_2.ipynb**, **task_2_3.ipynb**, **task_3_1.ipynb**, **task_3_2.ipynb**: Load `train_data.npy` and `test_data.npy` from `partB/data/` and use them for training the baseline and Latent SVM, evaluation (NDCG@5, P@5, MAP), ablations, and failure-mode analysis. No other datasets are used in the Part B pipeline.

## Files in this folder

| File            | Description |
|----------------|-------------|
| `train_data.npy` | List of per-query dicts with `query_id`, `X` (features), `y` (relevance). Generated by Task 2.1. |
| `test_data.npy`  | Same structure as train; test queries only. |
| `scaler.npy`    | `StandardScaler` fitted on training features (saved for reproducibility). |

To regenerate everything, run all cells in `partB/task_2_1.ipynb` from the `partB/` directory.
